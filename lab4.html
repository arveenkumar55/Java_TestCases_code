Background

You are working for a company (jBlink Software inc.) that recently had one of the programmers leave. The programmer was working on a library of methods in Java to support Statistical claculations.  The library was designed using generics and it is expecte to work with the Integer, Double and Long data types. 

The code appears complete and includes a main method that executes each of the public methods in the class with a using different supported data types. Since these are general purpose methods each has been declared static (all information required to accomplish the intended functionality is passed as method parameters). Your boss has asked you to do all of the following:

  • Set up a project using NetBeans and confirm operation of the program.
  • Place the code into a source control system. Specifically, you have been asked to use Git for this project. Git is included within Netbeans. A tutorial on how to use the features of Git required for this lab is available at Git NetBeans Tutorial
  • Document the all of the methods according to the accepted java standards as published at http://geosoft.no/development/javastyle.html.
  • Produce a set of unit tests that confirms that all of the public methods operate as expected.  The unit tests must attempt to use different data sets to ensure 100% code coverage and significant data coverage. 
  • Write-up a report summarizing your findings and include suggestions on improving the software development process at jBlink.

Tasks

You have been left with only the java source code for the project here

For this project follow the steps below:

  1. Create a new NetBeans project and add the source file to the project.
  2. Execute the project and ensure you see the results in the output window
  3. Initiate a GIT repository for the project, and make an initial commit - "COMMIT # 1 - Initial Commit". This initial commit should occur once you have a successful running program, but before you make any source code changes.  Esnure that you set your name as the contributor to the project.  You must use your Mohawk College email address within the submission.
  4. Document and improve the source code using the rules as provided in the Java Style Guideline. In order to understand the methods it is suggested that you modify the main method and attempt to pass different parameters to the methods. Specifically:
    • Improve the method names to reflect the purpose as you have determined.
    • Improve the method parameters to reflect the actual data requirements of the method.
    • Improve the variable names to reflect more accurately the role of the variables within each method.
    • Add comments to the code within the methods to describe the method functionality. It is important at this stage to document the inner working of the methods using standard comments. Use // comments for this step
    • Add documentation to each of the methods by adding full Javadoc comments to describe each of the methods and the purpose of the method. The javadoc must include @param, @return and @throws where applicable.
    • Once this is completed, commit these changes to the repository - "COMMIT#2 - Source Code Improvements".
  5. You must create unit tests as demonstrated in lecture and lab for each of the public methods provided. Ensure that your tests verify all of the intended functionality of the methods. You need to create tests that confirm normal operation (test to pass) and confirm any exceptions(test to fail) that are explicitly thrown in the class.  You do not need to test for NullPointerExceptions. Netbeans includes Version 4.x of jUnit which must be used for the tests. See the following tutorial for guidance on creating your tests http://netbeans.org/kb/docs/java/junit-intro.html#Exercise_42. For each method ensure the following:
    • Your tests provide 100% code coverage
    • Your confirm a variety of return values
    • Each unit test should use different input data that does not duplicate a previous unit test. Tests that do not provide additional value (prove something new) are considered duplicates.
    • Test boundary conditions where possible
    • Any method that returns a value that does not match the expected result, must fail. These failed cases will be the BUGS in the source code
  6. Commit these changes to the repository - "COMMIT#3 - Unit Tests added".

Report

You are to write a word document to summarize your findings and submit this along with the full NetBeans project source code that was constructed for the lab.

  • A title page including your name / student number and course name.
  • Include a statement of authorship at the bottom of the page (only include this on the title page). 
  • An introduction that explains the purpose of the report.
  • A summary of the development environment including hardware and software used.
    • Include the time required for you to understand the functionality of each of the methods.
  • A table that describes each of the methods in the class.
  • A list of the rules that you applied from the java style guideline to the code. You should list only the rules you actually applied to the code.  List the rules by section and number with a brief description of what the rule covers.
  • The list of Unit tests with a description of the test, why it is included, whether it passed of failed.
  • A summary of the bugs you found and a cross reference to the failed test(s) that proves the bug.
  • A list of what you feel are the most important improvements that you could make to the current development process. List up to three items in order of importance.

The report should not contain spelling or grammar errors, so feel free to make use of the Communications Center's proofreading service.
